Придумайте текстовый формат файла для визуализатора ДРАКОН-схем (по методологии Паранджанова) и кодогенерации из диаграмм, соответствующий следующим критериям:
Визуализатор будет сам решать где размещать иконки

1. Структура файла:
- Должен поддерживать хранение нескольких функций/процедур в одном файле
- Иметь секции для:
  * Header - произвольный код перед функциями
  * Код функций
  * Footer = произвольный код после функций
- Поддержка метаданных (автор, версия, зависимости)

2. Синтаксис должен:
- Сочетать человекочитаемость и машиночитаемость
- Использовать лучшие практики из YAML, TOML и современных конфигурационных форматов

3. Специфика ДРАКОН:
- Описание узлов схемы с поддержкой:
  * Стандартных элементов (действия, ветвления, циклы, силуэт)
  * Нового элемента "сворачиваемый блок" с:
    - Иконкой-индикатором (+/-)
    - Возможностью именованного описания свернутого содержимого
    - Атрибутами видимости по умолчанию

1. Предложите 2-3 варианта синтаксиса с обоснованием выбора
2. Приведите пример файла с:
   - Двумя функциями
   - Одним свертываемым блоком
   - Метаданными
   - if
   - switch
   - goto
3. Опишите механизм работы сворачивания в контексте:
   - Хранения состояния (свернуто/развернуто)
   - Визуального представления
   - Обработки при экспорте
